# Element Plus Admin UI

## 执行目标
- 快速构建可维护的后台 CRUD 界面。
- 统一交互细节，保证常见操作（查、增、改、删）路径清晰。
- 控制样式复杂度，优先可读与稳定。

## 推荐页面骨架
1. 构建“筛选表单区 + 操作按钮区 + 数据表格区 + 分页区”四段式布局。
2. 使用 `el-form` 承载筛选条件，使用 `el-table` 展示列表。
3. 使用 `el-dialog` 承载新增/编辑，避免跳转页面打断流程。
4. 使用 `el-pagination` 管理分页参数并联动查询。

## 交互规则
- 点击“查询”时重置页码为第一页，再发起请求。
- 点击“重置”时清空筛选条件并触发重新查询。
- 点击“删除”前给出确认框，成功后刷新当前页数据。
- 提交表单前做前端校验，提交成功后关闭弹窗并刷新列表。

## 代码约束
- 将查询参数与分页参数集中在同一响应式对象中管理。
- 将“拉列表”“保存”“删除”等请求方法拆分成独立函数。
- 将 loading 状态绑定到表格或关键按钮，避免无反馈等待。
- 将样式限制在当前组件，优先使用简洁 SCSS 而非深层覆盖。

## 完成检查
- 检查空列表、加载中、请求失败三种状态展示合理。
- 检查分页与筛选联动正确，不出现参数错位。
- 检查弹窗关闭后表单状态按预期重置。
- 检查按钮权限或禁用态（如有）符合业务条件。
